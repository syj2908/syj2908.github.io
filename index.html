一、主存储器概述

1. 存储器的分类

按作用分$\begin{cases}内部存储器\begin{cases}cache（主板上）\主存（“内存条”）\end{cases}\外部存储器\离线存储器\控制存储器（控制单元CU的核心）\end{cases}$

由于cache是主存内容的副本，因此内存容量只包括主存容量。

按存储介质分$\begin{cases}半导体存储器（主要做内部存储器）\此表面存储器\光盘存储器\end{cases}$

按存取方式分$\begin{cases}随机存储器RAM：任何单元内容能被随机存取且存取时间与物理位置无关\只读存储器ROM：非易失、高可靠，常作为主存一部分存放系统程序\顺序存取存储器SAM：如磁带📼\直接存取存储器DAM：工作方式类似RAM+SAM，如磁盘存储器\相联存储器AM\end{cases}$

按制造工艺分$\begin{cases}双极性存储器BJT:速度快、功耗高、价格贵，常作cache\金属氧化物半导体存储器FET：集成度高、价格低、功耗低、速度慢，常作主存\end{cases}$

按信息的可保存性分$\begin{cases}电易失型半导体存储器：RAM\非电易失型半导体存储器：ROM+部分RAM\end{cases}$

RAM分类$\begin{cases}SRAM：静态随机存储器。速度快、信息稳定、集成度低、功耗大、成本高，常作小容量cache。\DRAM：动态随机存储器。集成度高，功耗低，电容存在漏电现象，需要定时充电，称为“刷新”。常作主存。\NVRAM：非易失型随机存储器。SRAM+E^2PROM，有一个电池🔋保证掉电后信息得以保存。\end{cases}$

2. 主存储器与CPU的连接

（1）地址总线

地址总线位数决定可访问存储单元的最大数目（最大可寻址空间），即$0$~$2^n-1$。

（2）数据总线

数据总线位数决定了主存一次可读写的最大位数，也成存储字长。

（3）控制总线

CPU通过控制总线向主存发出操作命令。

3. 主存的性能指标

（1）存储容量

按字编址：存储单元个数×存储字长，单位b（bit），如1K×16位。

按字节编址：存放的字节数，单位B（Byte），如1KB。

（2）存取速度

存储周期=存取时间+恢复时间。

存储周期：连续访问两次存储器的时间间隔。

存取时间：一次存储器读写操作从启动到完成所需要的时间。

（3）存储器带宽

单位时间内存储器存取的信息量。谈论速度时，1K=1000。

存储器带宽=每个存取周期可访问的位数/存取周期。

半导体存储器

1. 主存储器的基本构成

$\begin{cases}存储体\寻址系统：驱动器+译码器+MAR\读写系统：读写电路+MDR。MDR存放要写的或要读出的数据\控制电路\end{cases}$

读操作过程：给出地址👉给出片选信号和读信号👉保存读出数据

写操作过程：给出地址：给出片选信号和待写入数据👉给出写信号

译码器地址译码$\begin{cases}线选法\重合法\end{cases}$

  • 线选法（单译码结构）

    线选法直接选中一个存储单元（如1K×8位的存储器，线选法地址线有10根，每次直接选中8位，即一个存储单元）。

    结构简单、速度快、器件用量大、成本高。

  • 重合法（双译码结构）

    重合法将地址码分成行地址和列地址，分别由行列译码器译码，重合点即为选中的存储元。

2. SRAM

SRAM是双稳态触发器。静态：只要保持电源供电，SRAM就能稳定保存数据。

SRAM写入时要双边选，读出时可单边读。

SRAM是非破坏性读出。

读写时序：①给地址信息👉②给输出/输入使能、片选使能👉③给出要读写的数据

读出时间：从地址线上给出有效地址到数据线上出现有效且稳定的数据所需要的时间。

读周期=读出时间+回复时间；写周期=写入时间+回复时间。

3. DRAM

由MOS管和电容组成，通过电容是否充电来存储0/1。

电容中电荷会慢慢泄露，因此要定时充电，称为“刷新”。

DRAM常做主存，由集成度高、功耗低、速度慢的特点。

从DRAM中直接读出的数据和存储数据是反向的，因此还需要增加反向电路。存储数据是同向的。

DRAM是破坏性读取，读取后要进行再生操作，一般要求读出放大器是再生放大器。

刷新$\begin{cases}集中式刷新\分散式刷新\异步式刷新\end{cases}$

刷新均为按行刷新。

  • 集中式刷新

    将刷新工作集中在一段时间内一次完成。会有“死区”产生。

    计算刷新时间时，每个最大刷新间隔内共有$n*t$的时间用来刷新，n指存储体行数，t指刷新一次的时间。

  • 分散式刷新

    将刷新分散到每个存取周期完成。存在重复刷新的情况，降低运行效率。

    计算刷新时间时，相当于存储周期翻倍（新周期内一个周期用来存取，一个周期用来刷新）。

  • 异步式刷新

    将每行的刷新平均到一个最大刷新周期间隔内完成，使得一个间隔内，每行有且仅有一次刷新。

    计算刷新时间时，直接用最大刷新周期间隔÷存储体行数。

DAM采用双译码结构，行列地址分时使用一组地址引脚，同时引入行地址选通信号$\overline{RAS}$和列地址选通信号$\overline{CAS}$。

DRAM读写时序：①提供行地址👉②$\overline{RAS}$有效👉③提供列地址👉④$\overline{CAS}$有效👉⑤在这之前读写使能信号有效👉⑥输出使能有效

4. 只读存储器和闪速存储器

ROM的主要用途是实现了软件固化。

ROM分类$\begin{cases}MROM掩模式ROM\PROM可编程ROM\EPROM紫外线擦除可编程ROM：浮栅有电荷“0”，无电荷“1”\E^2PROM电可擦除可编程ROM\end{cases}$

闪存：类似$E^2PROM$，速度快、成本低、容量大、功耗低、可在联机状态下擦除改写。

5. 存储容量的扩展及其与CPU的连接

$\begin{cases}字长不够：位扩展\容量不够：字扩展\都不够：字位扩展\end{cases}$

  • 位扩展

    各个存储器芯片的地址线、片选线、读写控制线分别按同名端并连起来，每个芯片的数据线分别引出作为整个存储器的数据线的一部分。

  • 字扩展

    各个存储器芯片的数据线、读写控制线分别并联，存储器地址线分为高位和地位，低位直接与各芯片的地址引脚相连，高位输入到片选译码器产生个芯片的片选信号。

  • 字位扩展

    综合上述两种方法进行扩展。

DRAM和SRAM扩展区别

  • DRAM地址线采取多路复用技术，需要动态存储控制器DRAMC提供。
  • DRAM不设片选引脚，用行地址选通信号代替。
  • DRAM需要定时刷新

DRAMC包括：

  • 刷新地址计数器
  • 地址多路选择器
  • 刷新定时器
  • 仲裁电路
  • 定时发声器

计算机系统简介

计算机系统=硬件+软件

硬件：组成计算机各种实际装置的总称

软件：具有各类特殊功能的程序和相关数据

软件又可分成系统软件和应用软件

系统软件：标准程序库、编译器、汇编器、操作系统、服务程序、数据库管理系统、网络软件等

软件和硬件在逻辑上是等效的

计算机系统的发展与应用

存储程序思想宣告了现代计算机结构思想的诞生

计算机硬件的组成

冯·诺依曼计算机的特点：

  • 五大部件：运算器、控制器、存储器、输入设备、输出设备
  • 指令和数据均由二进制形式表示
  • 指令和数据以同等地位存放在存储器中，可按地址访问
  • 指令由操作码和地址码组成
  • 指令在存储器内按顺序存放
  • 机器以运算器为中心，IO设备与存储器间的数据传送通过运算器完成

CPU=运算器+控制器

主机=CPU+主存储器（注意没有硬盘，它属于IO设备）

主存储器：可以直接与CPU交换信息、用来存放正在运行的程序和数据

存储元件->存储单元->存储体

每个存储元件能存放一位二进制代码；每个存储单元能存储一串二进制代码，称之为存储字，二进制代码的位数称之为存储字长

读/写操作以存储单元为最小访问单位

主存中的MAR&MDR：

  • MAR（Memory Address Register）：用来存放欲访问的存储单元的地址。位数和存储单元的个数有关，存储单元越多，位数越长。
  • MDR（Memory Data Register）：用来存放从存储器单元取出的代码或准备向某存储单元存入的代码，位数与存储字长有关。
  • MAR和MDR集成在CPU中。

控制器执行阶段：取指阶段->分析阶段->执行阶段，通过不同阶段来区分代码是指令或是数据

控制器=程序计数器PC+指令寄存器IR+控制单元CU

  • PC存放下一条指令在主存单元中的地址，与MAR间有直接通路，且有自动+1功能。
  • IR存放当前正在执行的指令代码，内容来自主存的MDR。
  • IR中的操作码送至CU进行指令分析。

运算器的核心是算术逻辑运算单元ALU，除此之外还包含一些寄存器，用来暂存运算数据和结果。

寄存器包括累加器AC、乘商寄存器MQ、操作数寄存器X

计算机系统的性能

性能=f(功能，质量)

机器字长指CPU依次能处理的二进制数据的最大位数。

存储字长=机器字长

指令字长=n*字节，n∈整数

内存=主存+cache

  • 按字编址，主存容量=存储单元个数×存储字长，单位为字（W）或位（b），如64K×32位。
  • 按字节编址，存储容量等于存储的字节数。

一、概述

指令系统是一台计算机所具有的全部机器指令的集合，由操作码和地址码组成。

指令$\begin{cases}宏指令\机器指令\微指令\end{cases}$

二、指令系统的发展

CISC

用一条复杂的新指令代替原先需要一串指令完成的功能，减少访问主存的次数以提高速度。

指令系统庞大复杂，大多数复杂指令很少用到。

RISC

只包含使用频率高的少量指令，并提供一些必要指令以支持高级语言。

  • 优先选取使用频率高的简单指令
  • 指令长度固定、格式种类少、寻址方式种类少。各字段划分较一致，功能也比较规整。
  • 只有LOAD\STORE可以访问存储器，其余在寄存器操作。
  • 有大量通用寄存器。
  • CPU采用流水结构。
  • 控制单元设计以硬布线控制逻辑为主。
  • 采用编译优化技术。

三、指令系统的功能

设计原则

  • 完整性
  • 规整性
    • 对称性
    • 匀齐性
    • 一致性
  • 高效性
  • 兼容性

数据类型

操作数类型

数据表示：机器硬件能够直接识别、指令能直接操作的数据结构

  • 地址
  • 数字
  • 字符
  • 逻辑数

字节或半字数据加载到32位寄存器时要扩充，有符号数高位用符号位填充，无符号数用0填充。

地址空间

地址空间$\begin{cases}通用寄存器\主存储器\IO设备\end{cases}$

  • 三地址空间

  • 两地址空间

    主存地址高端划分出部分作为IO设备地址。

    能够访问主存的指令都能访问IO设备，但会影响速度。

  • 一地址空间

    最低端是通用寄存器，最高端是IO设备地址，中间是主存。

编址方式

  • 按字编址

  • 按字节编址

    CPU可按字或字节访问主存。

字节编址顺序$\begin{cases}大端（低字节高地址）\小端（低字节低地址）\end{cases}$

边界对齐$\begin{cases}对齐：访问字节半字单子可在一个周期内完成\不对齐：节省空间，但访问速度可能低\end{cases}$

四、指令格式

指令组成

指令=操作码+地址码

操作码提供：指令操作类型和操作数类型

三种CPU结构$\begin{cases}堆栈型\累加器型\通用寄存器型(RR、RS)\end{cases}$

指令字长

指令字长：包含二进制码的位数

基本原则：

  • 指令字长尽量短
  • 信息为利用率尽量高

登场指令字结构、变长指令字结构

操作码扩展技术

等长指令字结构中，利用地址码个数较少的指令空出的地址码字段来增加操作码的位数。

一般方法：先安排地址码最多的指令。

五、寻址方式

$\begin{cases}指令寻址\数据寻址\end{cases}$

形式地址：符号地址。

有效地址：内存在本程序中的相对地址。

指令寻址

$\begin{cases}顺序寻址\跳跃寻址\end{cases}$

数据寻址

基本寻址

$\begin{cases}立即寻址\寄存器寻址\存储器寻址\堆栈寻址\end{cases}$

1.立即寻址

操作数直接在指令的地址码字段给出（立即数）。

$eg:MOV EAX, 100$

2.寄存器寻址

操作数已经在寄存器中，或者操作结果要存回通用寄存器中。

$eg: MOV EAX, 100$

3.存储器寻址

$\begin{cases}直接寻址\存储器间接寻址\寄存器间接寻址\偏移寻址\段寻址\end{cases}$

① 直接寻址

指令中地址码字段就是操作数或下一条指令的有效地址。

$eg：MOV EAX, [100]$

② 存储器间接寻址

指令中地址码字段是操作数地址的地址。

间接寻址一次称为一次间接，可以有多次间接。

间接寻址有一个标志位$\begin{cases}=0：后跟的地址码即有效地址\=1：后跟的依旧是地址的地址，继续套娃\end{cases}$

第一个间接寻址标志由指令给出，剩下的由访问主存所取出的地址码给出（通常是第一位）。

一次间接寻址的最大寻址范围是$2^m$字，两次间接寻址的最大寻址范围是$2^{m-1}$字。

$eg: JMP word ptr [100]$

③ 寄存器间接寻址

形式地址给出的是通用寄存器的编号。

对于一次寄存器间接寻址，寄存器中的内容就是有效地址。

寄存器间接寻址的范围取决于寄存器位数（也等于机器字长）。

$eg: MOV EAX, [EBX]$

④ 偏移寻址

形式上：直接寻址+寄存器间接寻址

$EA=(R_i)+A$，其中A是有符号数，通常用补码表示。

$\begin{cases}相对寻址\变址寻址\基址寻址\end{cases}$

i. 相对寻址

$EA=(PC)+A$

寻址范围：$(PC)-2^{k-1}$~$(PC)+2^{k-1}-1$

指令间相对距离不变，有利于程序在内存中浮动。

ii. 变址寻址

$EA=(R_i)+A$

变址寄存器中存放地址的修改量，每次会自动调整，形式地址给出地址的起始量。

寻址范围$2^n$，n是变址寄存器的位数。

iii. 基址寻址

基址寻址通常用于实现操作系统对用户程序的动态定位。

变址寻址与基址寻址异同：变址寻址重在偏移量的改变；基址寻址重在起始地址的不同。

⑤段寻址

段寻址是硬件自动完成的，过程对用户透明。

段寻址方式中的段内偏移量就是EA。

4. 堆栈寻址

利用堆栈存取数据。指令长度短，但运算速度低。

复合寻址

关键问题是计算顺序。

一、网络分层

1. 分层思想

每一层的目的都是向他的上一层提供服务。

$模板\begin{cases}①分层\②每层提供的服务\end{cases}$

$实例\begin{cases}①层与模板的对应关系\②每层协议\end{cases}$

分层思想的优势：

  • 各层之间相互独立
  • 灵活性好
  • 结构上可分割
  • 易于实现和维护
  • 能促进标准化工作

接口：用于定义下层向上层提供的原语操作和服务。

网络体系结构：网络分层+协议的集合

协议栈：为实现某一特定功能的相关协议集合。

2. 实体和协议

实体：层中的活动元素，包括软件和硬件。

对等实体：不同机器（发送端与接收端）里处于同一层次的实体。对等实体利用协议通信。

只有物理媒介执行的是真正的物理通信，对等实体间执行的都是虚拟通信。

不同网络层次中的实体实现的功能互不相同。

协议实现的细节和接口的描述不是体系结构的内容。

网络协议：为网络执行数据交换而建立的规则、标准或约定。包含三个要素：语法、语义、同步。

  • 语法：传输信息应遵循的数据结构形式。
  • 语义：对各个协议元素含义的解释。
  • 同步：规定实体之间通信操作执行的顺序。

3. 接口和服务

每一层的功能都是为他的上层提供服务的。功能$\neq$服务，服务是上层能看到的部分。

服务是在服务访问点（SAP）提供给上层使用的。

SAP总是属于下层的，即（n+1）层的IDU经过（n）层的SAP进入到（n）层。

4. 数据单元

数据单元：网络中信息传送的单位。

对等实体在协议下传送数据单元。

相邻层实体按服务传送数据单元。

$\begin{cases}协议数据单元PDU\接口数据单元IDU\服务数据单元SDU\end{cases}$

  1. PDU

    不同系统的对等实体为实现该层协议所交换的数据单元。

    PDU=PCI+UD

    PCI-协议控制信息

  2. IDU

    同一系统的相邻层实体的一次交互中，经过层间接口的数据单元。

    IDU=ICI+ID

    ICI-接口控制信息

  3. SDU

    上层服务用户所要求传递的数据单元，(n)接口数据的总额。

  4. (n+1)PDU=(n)SDU=(n)UD

    对于某一子层，接收的数据是SDU，处理后（分段/合并）发出的数据是PDU。

    PDU经过SAP时可能会分成多个IDU经过。

5. 面向连接服务和无连接服务

从数据通信的关联角度看，下层向上层提供的服务分成两大类，面向连接服务和无连接服务。

a. 面向连接服务

面向连接服务必须先建立连接再传输数据，具有连接建立、数据传输、连接释放三个阶段。

数据传输是按顺序传输的，易控制。

面向连接服务的保障强于无连接的。

b. 无连接服务

两个实体通信前无需预先建立连接，通信时不要求两个实体同时处于活跃状态。

无连接服务方便灵活，但不能防止数据丢失、重复、乱序。

$\begin{cases}面向连接服务适合在一定期间内向同一目的地发送多个数据的情形\无连接服务适合传输少量数据的情形\end{cases}$

无论那种服务，都不能保证数据传输的可靠性。

面向连接的可靠性服务$\begin{cases}消息流：保持应用的边界\字节流：没有消息边界\……\end{cases}$

6. 服务原语

服务在形式上是由一组服务原语来描述的。

服务原语只是概念性的功能描述，对于具体实现不作规定。

​ $\boxed{n+1}$

$\stackrel{请求、响应}{\downarrow}\quad\stackrel{指示、证实}{\uparrow}$

​ $\boxed{n}$

根据服务使用的服务原语类型，可将服务划分成有证实的和无证实的。

一种服务是否是证实的并非绝对，它依赖于网络层次提供的服务质量需求。

是否提供证实与最终结果是否是用户所期待的无关😂。

7. 服务和协议间关系

  • 服务只定义接口不涉及具体实现

  • 协议定义具体规则

  • 实体利用协议实现服务定义的接口

服务涉及上下相邻层间接口，协议涉及不同机器上对等实体间发送的数据分组。

二、网络参考模型

1. OSI参考模型

OSI参考模型本身并不是一个真正的网络体系结构，它只给出了每层应该实现的功能。

a. 物理层

在通信信道上透明地传输比特流。

物理层是一个逻辑概念，并不包含具体的物理介质。物理传输介质在物理层的下面，只有他们才进行了真正的物理传输。

b. 数据链路层

在物理线路上提供可靠的数据传输，使之对网络层呈现一条无错的线路。将原始的传输设施转换成一条逻辑的传输线路。

转换方法：数据分装成数据帧

任务$\begin{cases}寻址\链路管理\差错控制\流量控制\begin{cases}传输速率控制\缓存控制\end{cases}\成帧、帧同步\end{cases}$

c. 网络层

在源节点和目的节点间建立、维护、终止网络连接。控制通信子网的运行过程。

关键：如何将分组从源节点路由到目的节点。

任务$\begin{cases}统一编址\路由选择\拥塞控制\网络互连\end{cases}$

d. 传输层

为源主机到目的主机提供可靠、满足服务质量要求的数据传输服务。

传输层是真正端到端的层次。1-3层是点到点的（协议存在于每台机器与他的直接邻居之间），4-7是端到端的（协议存在于源主机和目的主机之间）。

任务$\begin{cases}屏蔽通信子网的实现\弥补要求的差异\提供进程级的访问（端口）\end{cases}$

e. 会话层

建立、引导、释放会话连接。

同步。

f. 表示层

对源端内部数据结构进行编码，形成适合网络传输的数据流，到目的端进行解码，形成目标端用户所要求的格式。

g. 应用层

为用户提供网络资源的访问服务。

2. TCP/IP参考模型

按需定制。

a. 网络接口层

对应物理层和数据链路层

没有定义具体的网络接口层协议，严格来说不是一个层次，只是一个接口。

b. 网际层

核心：IP协议

控制通信子网提供点到点的IP包传送服务。

无连接、不可靠但尽力而为的数据报传输服务。

c. 传输层

核心：TCP、UDP协议

提供端到端的数据传输服务。

TCP提供可靠的端到端的传输服务。

UDP提供不可靠的端到端的传输服务。

d. 应用层

提供面向用户的网络服务，对应OSI的高三层。

三、五层网络参考模型

  • 应用层
  • 传输层
  • 网络层
  • 数据链路层
  • 物理层

计算机网络的定义

“以能够相互共享资源的方式互连起来的自治计算机系统的集合。”

特征：

  • 资源共享
  • 分布在不同地理位置的独立“自治计算机”
  • 遵循共同的网络协议

判断计算机是否互联成计算机网络，主要看他们是不是独立的”自治计算机”

分布式系统重点在于系统，而非具体的某一台主机。

计算机网络发展历程

  • 以单计算机为中心的联机系统（第一代计算机网络）

    • 特征$\begin{cases}单处理机为中心的联机网络\集中式控制\end{cases}$
    • 主机负荷重
    • 通信线路利用率低
    • 可靠性低

  • 多点通信线路能够提高信道的利用率

    • 采用终端集中器/前端处理器处理终端通信任务

  • 分组交换网络（第二代计算机网络）

    • 主机互联方式有①主机直接互联②利用CCP（Communication Control Processor）间接互联
    • 采用分层思想，CCP构成通信子网（内层，数据传输），主机构成资源子网（外层，信息处理）
    • 多主机为中心，“主机——主机”连接方式
    • 划分分组、先存储再转发
    • ARPANET特点：①资源共享②分散控制③分组交换④采用专门的通信控制处理机⑤分层的网络协议

计算机网络体系结构标准化

协议是同等层次实体之间进行信息交互的规则。

计算机网络的体系结构=计算机网络的层次结构+各层协议的集合。

国际标准：OSI标准

实际标准：TCP/IP体系结构

网络拓扑结构

辨析：$\begin{cases}
传输介质（物理）\
传输信道（逻辑）
\end{cases}$

  • 星型结构

    广播/点到点均可

    建网容易，易于扩充，故障不扩散

    集中控制，对中心节点依赖性大，布线困难

  • 树形结构

    属于分层网络，有一定容错能力

    集中式控制，对上层节点依赖性强

  • 总线型结构

    线路长度短、布线容易

    故障有扩散性且定位困难

  • 环形结构

    主要是光纤网络

    数据在环上单向流动

  • 不规则形结构

    可靠性强，设置冗余通信线路以保证连通性

    传输效率低

  • 全连接型结构

计算机网络分类

按距离：广域网WAN（Wide Area Network）、局域网LAN（Local Area Network）

划分原因：连接节点数不同导致传输方式不同

按通信传播方式：广播网络、点对点网络

  • 广播网络：共享一个公共通信信道，一人发送数据所有人能接收到。但同时发送数据会产生冲突。一般用于小范围局域网。

    解决冲突：①“讲前先听”②“边讲边听”

  • 点对点网络：每条通信线路只链接一对计算机，依赖中间节点的存储转发，源端到目的端可存在多条路由，关键是路由选择。是否采用存储转发与路由选择是两种网络的重要区别之一。

操作系统学习随记（一）

发表于 2020-09-26 | 分类于 操作系统
字数统计: 1k | 阅读时长 ≈ 3

操作系统是管理计算机硬件的程序，它还为应用程序提供基础，并且充当计算机用户和计算机硬件的中介。

操作系统的功能

计算机系统自底向上可分为：硬件、操作系统、应用程序、用户。

从系统视角来看，可将操作系统看作资源分配器和控制程序。

操作系统是一直运行在计算机上的程序（Kernel）。

操作系统功能：进程管理、内存管理、文件管理、设备管理

计算机系统的组成

电源打开→运行引导程序→初始化系统各组件→定位操作系统内核并加载到内存。

CPU只能从内存加载指令。

每个计算机系统由一个CPU和多个设备控制器组成，他们通过共同总线连接在一起。每个设备控制器维护一定量的本地缓冲存储和一组特定用途的寄存器。设备控制器负责IO设备和本地缓冲存储之间的数据传递。操作系统为每个设备控制器提供设备驱动程序，该驱动控制设备控制器，并为操作系统其他部分提供访问接口。

开始IO→设备驱动程序加载设备控制器适当寄存器→设备控制器检查寄存器内容，决定下一步操作→控制器开始向本地缓冲区传输数据

IO处理$\begin{cases}同步处理\异步处理\end{cases}$

计算机系统的体系结构

  • 单处理器系统：只有一个通用CPU，可以有专用CPU
  • 多处理器系统：有两个以上紧密通信的CPU
    • 增加吞吐量
    • 规模经济
    • 增加系统可靠性
    • $\begin{cases}共享cache\共享IO接口\共享数据包\end{cases}$

操作系统的结构

$\begin{cases}脱机IO方式\简单批处理\多道批处理\分时系统\实时系统\end{cases}$

简单批处理特征：

  • 顺序性
  • 单道性
  • 自动性

多道批处理特征：

  • 多道性
  • 无序性
  • 调度性

实时系统要求有deadline，可硬可软。

在多道程序系统中，每个用户的作业都保存在硬盘（作业池）上，需要用的时候再转移到内存中。多道程序系统宏观上并行，微观上串行。

在分时系统中，作业直接进入内存，每个用户至少有一个程序在内存中。

多道程序系统和分时系统区别：多道程序系统重点在”同时“运行多个程序，而分时系统重点在同时为多个用户提供服务。

操作系统的执行

现代操作系统是中断驱动的。

双重模式——用户模式和内核模式：

  • 计算机硬件通过模式位表示当前模式。
  • 每当操作系统控制计算机时，它处于内核模式。当用户应用通过系统调用请求操作系统服务时，系统必须从用户模式切换到内核模式。
  • 可能引起损害的机器指令为特权指令，硬件只有在内核模式下才能执行特权指令。
  • 常见特权指令：切换到用户模式、IO控制、定时器管理、中断管理。

进程管理

程序是被动实体，进程是主动实体。

单线程进程有一个PC，多线程进程有多个PC。

进程是系统的工作单元，系统由多个进程组成。

内存管理

内存是现代计算机系统执行的中心。

内存一般是CPU所能直接寻址和访问的、唯一的、大容量的存储器。

存储管理

计算机运行的最终速度与硬盘子系统的速度和管理该子系统的算法由很大关系。

高速缓存是一个相对的概念，相对更快的存储系统对于相对更慢的存储系统来说都是高速缓存。

高速缓存要注意一致性。

  2. 新建一个本地文件夹作为仓库

基本流程：修改文件→提交改动到缓存→提交缓存到仓库

git add <file/dir>：将改动添加到缓存

git status -[s]：查看自上次提交之后是否有修改

git diff -[cached]：查看改动的详细信息

git commit -m "annotation"：将缓存提交到仓库

git rm <file>：取消跟踪文件

git mv <file>：文件重命名

本来教材选用的是《鸟哥的Linux私房菜》，讲的很详细，根据书上的指导成功搭建除了centos虚拟机，但越往后学习越觉得这本书作为教材“苏味儿”太浓了：详细全面并且啰嗦。每条命令光是十几条options就能把人看傻，因此看到一半，便挑了一本《Linux命令行大全》去看了。浅显、基础、简单，很适合初学者。但这本书并没有类似私房菜的前期虚拟机配置相关的指导，可能作者们大多数都默认了环境这个东西有手就行吧233
