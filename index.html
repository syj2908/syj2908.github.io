Git使用指北

发表于 2020-08-17 | 分类于 Git
字数统计: 889 | 阅读时长 ≈ 3

“Git 是 Linus Torvalds 为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件。”

最近在整理之前的各种资料，有时候需要把整个文件夹传到Github上，但是之前一直采取的是手动拖拽文件的上传方式，文件数量一多这个方法就会显得非常繁琐。因此便借着这个契机学学怎么用Git。

Git的工作流程

仓库初始化

  1. 获取ssh密钥并把它添加到你的GitHub账号上去
  2. 新建一个本地文件夹作为仓库
  3. 仓库初始化 git init 这条command会在文件夹中生成一个.git文件，用于保存仓库快照

这样一个本地仓库就搭好了。之前搭这个博客的时候已经完成了第一步，因此真正要做的事情并不多。

仓库基本管理

基本流程：修改文件→提交改动到缓存→提交缓存到仓库

git add <file/dir>：将改动添加到缓存

git status -[s]：查看自上次提交之后是否有修改

git diff -[cached]：查看改动的详细信息

git commit -m "annotation"：将缓存提交到仓库

git rm <file>：取消跟踪文件

git mv <file>：文件重命名

git add <file/dir>

比如你新建了一个文件foo，修改了一个文件bar，想要把这些改动提交到缓存，就命令行写入git add foo bar。

命令对象同样支持目录，一般新建一个仓库，要把所有文件都提交到缓存，就用git add .即可。

git status -[s]

可以查看有哪些改动提交到了缓存但还没提交到仓库。

git rm -[r f] <file/dir>

仅仅将一个文件shift+D删除是不够的，因为Git依旧会追踪这个文件的去向，要想完全将文件删除（或者说取消追踪）就要用到这条命令。其中参数-r是允许递归删除目录，-f允许强制删除/

git mv <file/dir>

普普通通的重命名。

Git+Github远程仓库

在对本地仓库commit好了后，就要将远程仓库更新。

添加远程仓库

在仓库文件夹中打开git bash，输入git remote [repo_name] [url]。

其中repo_name我一般喜欢和仓库名同名，url即Github仓库的url，这个指令每个仓库只用配置一次下次打开他会自动配置。

从Github仓库pull数据

git pull [repo_name] [branch_name]

push数据到Github仓库

git push [repo_name] [branch_name]

注意这里要指定分支名。

正确的关机📴姿势

由于Linux是多人多任务系统，因此不正确的关机可能会导致数据的丢失与损坏。

关机操作只能由root执行。

  • who：查看目前有谁在线
  • netstat-a：网络联机状况
  • ps-aux：查看后台执行程序
  • sync：数据同步写入硬盘指令，一般用户执行此指令只能把自己的数据同步写入，root执行此指令可以把所有用户的数据同步写入。每次关机以前一定要先同步数据。
  • shutdown -h now：立即关机
  • shutdown -r 10：10分钟后重启（如果不设定时间，默认一分钟后执行操作）
  • shutdown -k 'I will shut this system'：不关机，只是发送警告信息
  • shutdown -c：取消关机计划

Linux的文件权限与目录配置

查看所有文件详细权限和属性的命令：ls -al

文件属性

文件属性示意：-rw-r--r-- 1 root root 1864 May 4 16:01 initaial-setup-ks.cfg

  1. -rw-r--r--：文件类型权限，具体还能分成四部分

    • -：表示文件类型，d是目录，-是文件，|是链接🔗文件（相当于快捷方式），b是周边存储设备，c是串行端口设备
    • 剩下九个字符三个一组，分别表示的是文件拥有者的权限、加入用户组账号的权限、一般用户的权限。r表示可读，w表示可写，x表示可执行。rwx排列顺序固定，若没有某一权限，就会用-代替那个字符。比如例子中的rw-，表示文件拥有者对这个文件可读可写不可执行。

  2. 1：表示链接到这一节点的文件数量

  3. root：文件拥有者

  4. root：文件所属的用户组

  5. 1864：文件大小，默认Byte

  6. May 4 16：01：文件最后修改时间

  7. initaial-setup-ks.cfg：文件名

理论知识梳理

这本书前面粗略讲解到了许多计算机概论和操作系统相关的理论知识，鉴于大三也要学操作系统这门课，也就顺便梳理一下这部分比较重要的知识吧👀。

最常见的两种CPU架构

精简指令集（RISC）：指令功能简单，运行时间短。ARM公司的ARM CPU就是这种架构。

复杂指令集（CISC）：指令功能复杂，运行时间较长。Intel、AMD等x86架构的CPU使用的就是这种架构。

PC架构与相关设备组件

北桥 南桥控制主板各组件间的通信

北桥：连接🔗CPU、显卡、内存等速度较快的组件。现在一般都将内存控制器直接整合到CPU中，使CPU和内存直接交流，即提高了速度，也减少了占用的带宽。

南桥：连接🔗IO设备接口等速度较慢的组件，例如硬盘、USB、网卡等。

CPU

CPU的工作频率

外频：CPU和外部组件之间进行数据传输的速度

倍频：CPU用来加速工作性能的一个倍数

CPU工作频率=外频×倍频

超线程（HT）

每个CPU内将寄存器分为两组，让程序分别使用一组寄存器，不通过操作系统的多任务切换，而是让他们竞争使用CPU的资源，以此达到“仿佛同时有两个内核在工作”的效果。

内存

  • PC内存主要组件为DRAM（动态随机存取内存），DRAM又分为SDRAM和DDR SDRAM。
    DDR是指double data rate(双倍数据传输速度），因此传输频率方面比较优秀。
    类似的还有DDR2、DDR3，分别指频率倍数是4倍、8倍。

  • CPU内部的二级缓存使用的是SRAM（静态随机存取内存）

  • BIOS和固件存储在ROM中。

硬盘与存储设备

  • 扇区是最小的物理存储单位
  • 同一个同心圆上的扇区组合成的圆⚪就是磁道
  • 所有碟片上同一个磁道组成柱面
  • 硬盘内部机械臂的磁头和碟片之间有细微的空间，若有抖动或者污物附着就可能造成数据的损坏或者硬盘的损坏。并且要使用操作系统的关机方式，以保证机械臂正确归位

操作系统

  • 操作系统内核：管理电脑所有活动以及驱动系统中所有硬件，主要功能包括：
    系统调用接口
    进程管理
    内存管理
    文件系统管理
    设备驱动
  • 系统调用：为程序员提供一套应用程序编程接口

磁盘分区

磁盘的命名问题

所有设备在Linux系统下都是当作一个个文件来处理的，物理机中磁盘的名字一般为/dev/sd[a-p]，虚拟机中磁盘的名字一般为/dev/vd[a-p]。至于abcd的顺序则是根据系统检测到的磁盘顺序命名的。

磁盘分区格式

MBR（MS-DOS）格式

启动引导程序纪录区和分区表放在磁盘的第一个扇区，这个扇区通常是512字节。

  • 主引导记录：安装启动引导程序的地方，有446字节
  • 分区表：记录整块硬盘分区状态，有64字节

由于分区表的大小限制，我们默认最多只能分出四个分区，我们一般称为主要分区和扩展分区。扩展分区还能继续分区，分出来的就叫做逻辑分区。

逻辑分区的命名是从sd5开始的，前面四个代号是留给主要分区和逻辑分区用的（不管它们是否真的用完了）。

这种格式由于大小限制，最大只能记录2TB左右的数据。

GPT磁盘分区表（partition table）

GPT将磁盘所有区块以LBA（逻辑区块地址）来划分（默认512字节），一共用了34个LBA，分别叫做LBA0~LBA33，并且磁盘的最后34个LBA也拿来作了一个备份。

  • LBA0：功能类似MBR，存储了启动引导程序和一个特殊标志字符，以此表示是GPT格式。
  • LBA1：GPT表头部分，记录了分区表本身的位置和大小，同时记录了备份GPT分区的位置。另外还有校验码之类的东西。
  • LBA2~LBA33：实际的分区表。理论上说最大可以记录8ZB大小的数据。

计算机启动流程

  1. BIOS：执行的第一个程序，它会识别出第一个可启动设备
  2. MBR/LBA0：第一个可启动设备的第一个扇区的主引导记录块，还有启动引导程序代码
  3. 启动引导程序（boot loader）：一个可读取内核文件并且执行的软件，他会启动内核文件
  4. 内核文件：启动操作系统

启动引导程序的功能：

  • 提供不同的启动选项
  • 加载内核文件
  • 转交其他启动引导程序（多重引导，让你可以选择是用Windows还是Linux操作系统）

每个分区的都有启动扇区，启动扇区里也可以安装启动引导程序。每个启动引导程序只会认识自己分区的内核文件以及其他的启动引导程序。

关于挂载

挂载，就是利用一个目录当作入口，将磁盘分区的数据放在该目录下。也就是说，进入该目录即可访问该分区的数据。

有穷自动机

DFA

A=(Q,Σ,δ,q0,F)

  • Q：状态集合
  • Σ：输入符号集合
  • δ：转移函数
  • q0是起始状态
  • F是接受状态

NFA

转移表中每一项都是一个集合，即使只有一个可达状态也是一个集合，没有可达状态就是空集。

NFA转DFA——子集构造法，惰性求值

ε-NFA转DFA与一般NFA转DFA的区别是：DFA开始集合是ε-NFA开始集合的ECLOSE。

求ECLOSE(q)

从q出发寻找所有ε可达的符号，将其加入ECLOSE(q)，然后对新加入的符号重复这一操作，直到ECLOSE(q)不再变大。

DFA最小化

  • 状态等价：对两个状态输入任意合法的符号串，结果要么是都接受，要么是都不接受。

  • 填表算法：接受状态和非接受状态不等价，以此为基础对每对状态进行检验…

  • 证明两个DFA等价：将两个DFA的转移图看作一个DFA的转移图，对这个新的DFA进行填表法。若最终两个原DFA的起始状态等价，则证明这两个DFA等价。

  • DFA最小化：把所有等价状态归为一类。

范式

产生范式的步骤

  1. 去除可空产生式
  2. 去除单位产生式
  3. 去除无用符号

去除可空产生式

  1. 求可空集合
  2. 对于含有可空符号的产生式进行排列组合，由原来的产生式分裂成空/非空两类产生式

去除单位产生式

  1. 找单位产生式，求出所有单位对

  2. 对于每个单位对(A,B)，把所有的产生式A→B加入产生式集合P，其中B→α是P中的非单位产生式。（非形式地说，就是把所有单位产生式的“中介符号”消除掉）

去除无用符号（所有含有该符号的产生式都要去除）

  1. 有用的符号：产生的且可达的
    不是有用符号的是无用符号

  2. 判断有用符号：先判断产生的，再判断可达的

    • 求可生成集合

      • 每个终结符是产生的

      • 若有A→α且α中的符号都是产生的，则A也是产生的。（由后向前）

    • 求可达集合

      • 开始符号是可达的

      • 若A可达，所有以A为产生式头的产生式体中的符号也是可达的。（由前向后）

构建乔姆斯基范式

  1. 产生式只有两种形式

    • A→BC
    • A→α

  2. 方法：打断重排

    • 对于终结符，引入A→α使产生式要么全是非终结符，要么是前面这种形式

    • 对于较长只有变元的产生式，采取引入变元的方式将长产生式打断成短产生式

上下文无关文法定义

G=(V,T,P,S)

其中V是变元集合，T是终结符号集，P是产生式集合，S是初始符号集合。

推导、最左推导

  • 推导：由开始符号开始，通过产生式一步步得到目标字符串。
  • 最左推导：每次推导只替换最左边的变元。

下推自动机

PDA定义

P=(Q,Σ,T,δ,q0,Z0,F)

  • Q：状态的有穷集合

  • Σ：输入符号的有穷集合

  • T：有限的堆栈字母表

  • δ：转移函数

  • q0：初始状态

  • Z0：初始符号（堆栈中）

  • F：接受状态

转移函数

基本格式δ(q,a,X)=(p,γ)

表示当状态为q，输入字符为a，栈顶符号X时，转移到状态p，栈顶的X被替换成γ。

状态转移图

由q指向p：状态从q转向p

线上的内容a,X/α：输入符号是a，栈顶符号是X，用α替换X。

PDA的瞬时描述

三元组(q,w,γ)：q是当前状态，w是剩余的输入串，γ是堆栈内容。

以终结状态方式接受&以空栈方式接受

  1. 以终结状态方式接受
    当输入串消耗完成后，PDA恰好停留在一个接受状态，证明输入串被接受。

  2. 以空栈方式接受
    当输入串消耗完成后，PDA的栈恰好为空，证明输入串被接受。

  3. 空栈方式到接受状态方式
    在开始分析前先向栈底压入一个标记X0，当输入串消耗完后，若栈顶恰好是X0，则无条件转移到接受状态。
    为此，要新建一个开始状态用来把X0压栈，一个终结状态用来被无条件转移到这里。

  4. 接受状态方式到空栈方式
    为每个原接受状态建立一条空转移到新的终结状态，并且在这个过程中弹出栈中所有剩余的东西。

构建FIRST集合

  1. 对于终结符，FIRST集合只包含它本身。

  2. 对于非终结符，若有产生式X→a…，则把a加入到FIRST(X)中；若有X→ε，则ε同样也加入。

  3. 分三类

    • 若有X→Y…且Y是非终结符，则把FIRST(Y)中所有非空元素加入到FIRST(X)中；
    • 若有X→Y1Y2Y3…且Y1、Y2、Y3…是非终结符，若Y1~Yi-1的FIRST集都含有ε，则把FIRST(Yi)中所有非ε元素加入到FIRST(X)中；
    • 若有X→Y1Y2Y3…且Y1、Y2、Y3…是非终结符，若Yi的FIRST集都含有ε，则把ε加入到FIRST(X)中。

构建FOLLOW集合

  1. 对于开始符号，把#放到FOLLOW中。

  2. 若有A→αBβ，则把FIRST(β)/{ε}的所有元素加到FOLLOW(B)中。

  3. 若有A→αB，或A→αBβ而FIRST(β)中有ε，则把FOLLOW(A)中元素加到FOLLOW(B)中。

构建LL（1）分析表

对象：文法中的每个产生式A→α

  • 对于终结符a属于FIRST(α)，把A→α加到M[A,a]中。
  • 若ε属于FIRST(α),则对任何b属于FOLLOW(A)，把A→α加到M[A,b]中。

自下而上分析——LR(0)&SLR(0)文法

短语、直接短语、句柄

  • 短语：若有S→αAδ且A→β， 那么β是句型αAδ相对A的短语
  • 直接短语：A能一步推导出β
  • 句柄：最左直接短语。在语法树中，句柄是最左子树的端末结的自左至右排列，该子树有且只有两层。

LR分析法

问题

构造一种自下而上分析法，从而求出输入串的最左规约

核心：ACTION表&GOTO表

  1. ACTION表决定移进还是规约。若移进，ACTION告诉了移进后的状态；若规约，ACTION告诉了用哪条产生式规约。
  2. GOTO表决定规约后的新状态是什么。
  3. 规约时，注意符号栈和状态栈的长度始终相等，若规约串的长度为a，则要同时从符号栈和状态栈中弹出a个符号/状态，然后再移进规约后的新符号和新状态。

构建思路

构建拓广文法->构建CLOSURE闭包&GO函数->构建项目集规范族->构造LR(0)的ACTION表&GOTO表

具体做法

  1. 构建拓广文法：在原文法前加入一个新开始状态S’，目的是这样会有一个唯一的接受状态，S’→S·
  2. 构建CLOSURE闭包
    • I的任意项目都属于CLOSURE(I)
    • 若A→α·Bβ属于CLOSURE(I)，则对B的任何产生式B→γ，把B→·γ加入到CLOSURE(I)中
    • 重复上述操作直到闭包不再变大
  3. 构建GO函数
    • 直观理解：GO(I,X)=CLOSURE(J)就是在状态为I的时候，输入符号为X时转移到了状态J。然后再对J求闭包构成下一个状态。
  4. 构建项目族规范集
    • 从开始符号开始，先构建开始符号的CLOSURE(S’)闭包
    • 对每个文法符号X，构建GO(S’,X)
    • 对每个GO函数产生的新项目，构建其CLOSORE闭包
    • 重复上述过程，直到项目族规范集不再增大
  5. 构建LR(0)的ACTION表&GOTO表

SLR分析法

构造过程和LR分析法基本相同，唯一不同的是在判断规约动作A→α·（设置ACTION[K,a]）的时候，需要保证a属于FOLLOW(A)。
