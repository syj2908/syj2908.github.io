Linux 学习札记（一）

理论知识梳理

这本书前面粗略讲解到了许多计算机概论和操作系统相关的理论知识，鉴于大三也要学操作系统这门课，也就顺便梳理一下这部分比较重要的知识吧👀。

最常见的两种CPU架构

精简指令集（RISC）：指令功能简单，运行时间短。ARM公司的ARM CPU就是这种架构。

复杂指令集（CISC）：指令功能复杂，运行时间较长。Intel、AMD等x86架构的CPU使用的就是这种架构。

PC架构与相关设备组件

北桥 南桥控制主板各组件间的通信

北桥：连接🔗CPU、显卡、内存等速度较快的组件。现在一般都将内存控制器直接整合到CPU中，使CPU和内存直接交流，即提高了速度，也减少了占用的带宽。

南桥：连接🔗IO设备接口等速度较慢的组件，例如硬盘、USB、网卡等。

CPU

CPU的工作频率

外频：CPU和外部组件之间进行数据传输的速度

倍频：CPU用来加速工作性能的一个倍数

CPU工作频率=外频×倍频

超线程（HT）

每个CPU内将寄存器分为两组，让程序分别使用一组寄存器，不通过操作系统的多任务切换，而是让他们竞争使用CPU的资源，以此达到“仿佛同时有两个内核在工作”的效果。

内存

  • PC内存主要组件为DRAM（动态随机存取内存），DRAM又分为SDRAM和DDR SDRAM。
    DDR是指double data rate(双倍数据传输速度），因此传输频率方面比较优秀。
    类似的还有DDR2、DDR3，分别指频率倍数是4倍、8倍。

  • CPU内部的二级缓存使用的是SRAM（静态随机存取内存）

  • BIOS和固件存储在ROM中。

硬盘与存储设备

  • 扇区是最小的物理存储单位
  • 同一个同心圆上的扇区组合成的圆⚪就是磁道
  • 所有碟片上同一个磁道组成柱面
  • 硬盘内部机械臂的磁头和碟片之间有细微的空间，若有抖动或者污物附着就可能造成数据的损坏或者硬盘的损坏。并且要使用操作系统的关机方式，以保证机械臂正确归位

操作系统

  • 操作系统内核：管理电脑所有活动以及驱动系统中所有硬件，主要功能包括：
    系统调用接口
    进程管理
    内存管理
    文件系统管理
    设备驱动
  • 系统调用：为程序员提供一套应用程序编程接口

磁盘分区

磁盘的命名问题

所有设备在Linux系统下都是当作一个个文件来处理的，物理机中磁盘的名字一般为/dev/sd[a-p]，虚拟机中磁盘的名字一般为/dev/vd[a-p]。至于abcd的顺序则是根据系统检测到的磁盘顺序命名的。

磁盘分区格式

MBR（MS-DOS）格式

启动引导程序纪录区和分区表放在磁盘的第一个扇区，这个扇区通常是512字节。

  • 主引导记录：安装启动引导程序的地方，有446字节
  • 分区表：记录整块硬盘分区状态，有64字节

由于分区表的大小限制，我们默认最多只能分出四个分区，我们一般称为主要分区和扩展分区。扩展分区还能继续分区，分出来的就叫做逻辑分区。

逻辑分区的命名是从sd5开始的，前面四个代号是留给主要分区和逻辑分区用的（不管它们是否真的用完了）。

这种格式由于大小限制，最大只能记录2TB左右的数据。

GPT磁盘分区表（partition table）

GPT将磁盘所有区块以LBA（逻辑区块地址）来划分（默认512字节），一共用了34个LBA，分别叫做LBA0~LBA33，并且磁盘的最后34个LBA也拿来作了一个备份。

  • LBA0：功能类似MBR，存储了启动引导程序和一个特殊标志字符，以此表示是GPT格式。
  • LBA1：GPT表头部分，记录了分区表本身的位置和大小，同时记录了备份GPT分区的位置。另外还有校验码之类的东西。
  • LBA2~LBA33：实际的分区表。理论上说最大可以记录8ZB大小的数据。

计算机启动流程

  1. BIOS：执行的第一个程序，它会识别出第一个可启动设备
  2. MBR/LBA0：第一个可启动设备的第一个扇区的主引导记录块，还有启动引导程序代码
  3. 启动引导程序（boot loader）：一个可读取内核文件并且执行的软件，他会启动内核文件
  4. 内核文件：启动操作系统

启动引导程序的功能：

  • 提供不同的启动选项
  • 加载内核文件
  • 转交其他启动引导程序（多重引导，让你可以选择是用Windows还是Linux操作系统）

每个分区的都有启动扇区，启动扇区里也可以安装启动引导程序。每个启动引导程序只会认识自己分区的内核文件以及其他的启动引导程序。

关于挂载

挂载，就是利用一个目录当作入口，将磁盘分区的数据放在该目录下。也就是说，进入该目录即可访问该分区的数据。
