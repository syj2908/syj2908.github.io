Git使用指北

发表于 2020-08-17 | 分类于 Git | 热度 ℃
字数统计: 889 | 阅读时长 ≈ 3

“Git 是 Linus Torvalds 为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件。”

最近在整理之前的各种资料，有时候需要把整个文件夹传到Github上，但是之前一直采取的是手动拖拽文件的上传方式，文件数量一多这个方法就会显得非常繁琐。因此便借着这个契机学学怎么用Git。

Git的工作流程

仓库初始化

  1. 获取ssh密钥并把它添加到你的GitHub账号上去
  2. 新建一个本地文件夹作为仓库
  3. 仓库初始化 git init 这条command会在文件夹中生成一个.git文件，用于保存仓库快照

这样一个本地仓库就搭好了。之前搭这个博客的时候已经完成了第一步，因此真正要做的事情并不多。

仓库基本管理

基本流程：修改文件→提交改动到缓存→提交缓存到仓库

git add <file/dir>：将改动添加到缓存

git status -[s]：查看自上次提交之后是否有修改

git diff -[cached]：查看改动的详细信息

git commit -m "annotation"：将缓存提交到仓库

git rm <file>：取消跟踪文件

git mv <file>：文件重命名

git add <file/dir>

比如你新建了一个文件foo，修改了一个文件bar，想要把这些改动提交到缓存，就命令行写入git add foo bar。

命令对象同样支持目录，一般新建一个仓库，要把所有文件都提交到缓存，就用git add .即可。

git status -[s]

可以查看有哪些改动提交到了缓存但还没提交到仓库。

git diff -[cached]

可以查看具体是哪里有改动

git commit -m “annotation”

将缓存提交到仓库，必须要加一句注释

git rm -[r f] <file/dir>

仅仅将一个文件shift+D删除是不够的，因为Git依旧会追踪这个文件的去向，要想完全将文件删除（或者说取消追踪）就要用到这条命令。其中参数-r是允许递归删除目录，-f允许强制删除/

git mv <file/dir>

普普通通的重命名。

Git+Github远程仓库

在对本地仓库commit好了后，就要将远程仓库更新。

添加远程仓库

在仓库文件夹中打开git bash，输入git remote [repo_name] [url]。

其中repo_name我一般喜欢和仓库名同名，url即Github仓库的url，这个指令每个仓库只用配置一次下次打开他会自动配置。

从Github仓库pull数据

git pull [repo_name] [branch_name]

push数据到Github仓库

git push [repo_name] [branch_name]

注意这里要指定分支名。
